#001 Intro to HTML, CSS, Audio files and Local Web Storage

Thursday 8th September 2016

The first task was to create a blank HTML file called 'index', containing the basic structure that makes up a website, i.e <html>, <head>, <body> tags. The html tag tells us what language we are going to program in, then below that the head tag will contain any links or scripts needed for the page. The body tag will contain all the elements needed to make up the webpage. The next step was to add in a header, an introduction paragraph, a table of contents and a hyperlink to the W3schools website. The header is inserted using a <h> tag, followed by a number between 1 & 7 (determines the size of the header with 1 being the largest). The paragraph was inserted by using the <p> tag, telling the computer that we want to insert a paragraph.

The table is a bit more than just one tag as we need to tell the computer when to insert rows, column etc. It starts with a <table> opening tag, followed by a <tr>, telling us that we want to insert a row. Inside that tag, we use a series of <td> tags, which will determine how many columns we want. As the very first row in a table is usually a header, we can use <th> instead of <td> for the first row which will automatically format the first row into a header. To add in a hyperlink for w3schools, I decided to hyperlink a picture of the w3schools logo in the footer. To do this, I used an anchor tag with the link in ‘href’, i.e. <a href=“##”>, followed by the image before the ‘a’ tag was closed off. I then created an external css file, meaning that I could format the elements created, changing the font, size, colour and position. to format the colour in CSS, we simply use colour: ##; followed by the colour in hexadecimal or predetermined colours. To change the size of something, for example, my jumbotron, we can use height: ##; and width: ##; which allows us to specify the exact width and height in pixels, percentages, em or rem. The different sizing options will suit whether or not you prefer a fixed or fluid layout. The table and introduction paragraph we now want to be fluid layout. We do this by changing their size properties to percentages, and setting position to absolute.

Two audio files, one .mp3 and the other .ogg, were created by taking my audio recording and converting them into both audio formats on www.media.io. The next stage is to insert the audio and controls by using <audio controls>, which will display a default media player. The final element inserted was an API, more specifically a local web storage. I did this by creating a script, then using an if statement to determine if the computer is capable of storing data. If so, I used localStorage.setItem to set the user, title, date and content of the log. Once they are stored, we need to retrieve them and display it to the screen. To retrieve, we use document.getElementById(“##”).innerHTML = localStorage.getItem(“##”), where the getElementById will search for that id within the html code, and input the content we insert into .getItem(). In my website, I inserted a <div> with an id=“result” to display the stored content.

#004 Internet Architecture

Tuesday 20th September 2016

This unit is based around the Internet architecture and the various types of networks, bandwidths, protocols etc. that is needed to connect a client to the internet. A client server network is designed to allow the client access to resources from a central computer, i.e a server. The main purpose of this is that it acts as a hub for a number of less powerful computers. The hardware and software that a client needs to successfully connect to the internet is a PC or a device connected to a network card or modem, which will be connected to a router that sends and receives the data packets, and a suitable browser on the device so that it can interpret the data using protocols and display it to the screen.

The data is transmitted from the client to the server can vary in speed, in both upload and download times. This is relative to the bandwidth, which is the amount of data that can be transmitted at any given time (usually measured per second). The three types of bandwidth that we looked at was PSTN, ISDN and DSL. PSTN or Public Switched Telephone Network is basically the world’s circuit-switched telephone networks, sometimes referred to as the Plain Old Telephone Service (POTS). ISDN or Integrated Services Digital Network is used for sending voice, video and data over digital or normal telephone lines, which supports transfer rates of around 64 Kbps/s. DSL or Digital Subscriber Line was created to allow high bandwidth transfer rates across all homes and businesses using just normal telephone lines. The speed can range anywhere from 256kbit/s to over 100Mbit/s.

When data is transferred from server to the client, it will be just raw data which would be meaningless without protocols. There are various different types of Protocol, such as TCP/IP, HTTP, SMTP or POP3, which are made up of rules so that computers can interpret and read data. Transmission Control Protocol (TCP) is the process of breaking up information that has been sent into smaller pieces, called data packets. These packets are then sent and put back together in the correct order. Internet Protocol (IP) is linked with TCP as it is the final stage in taking the data packets and directing them to the required destination, using an IP address which will be unique to every computer. Hypertext Transfer Protocol or HTTP is most commonly known for being the protocol used for the World Wide Web. It is the process of using TCP/IP to transfer HTML, so that when a URL is entered into a browser, it will fetch and display the webpage. Simple Mail Transfer Protocol (SMTP) and Post Office Protocol version 3 (POP3) are used for the sending and receiving of electronic mail. SMTP is used mainly in the sending of emails, whereas POP3 is used along side to receive emails, as SMTP has a limited ability to queue incoming messages.

There is a process of how the internet works, and how we receive data transmissions. It begins with a PC or device that is linked to a router. If, for example, the client enters a URL into a browser, it will send a request to the closest Point of Presence, which is run by your Internet service provider. It will then send back the necessary data back to the router/modem. If, however, you require information from another country or ISP, it will then find the closest Internet Exchange Point (IXP), which will cover several thousand POP’s. It is then sent back to the POP, then back to the client. This process can result in longer loading times, but in reality, is a very efficient way of networks to exchange information, as well as reducing costs.

When a user has entered a domain to take them to a website, the IP address needs to be located. This is where Domain Name Systems (DNS) comes into play as it acts as an address book for IP’s, making it an essential component of the internet. The way that an IP address will be located is by the naming structure, i.e, the prefix (http) tells us that we want to use the Hypertext transfer protocol and that we are communicating with the World Wide web. The subdomain (www.) also tells us that we are searching the WWW. The domain name is the next part (example.com), which is the website unique identifier, as no two websites will have the same domain. The last part of the domain name (.com) is called the extension, which tells us what type of website is being run, e.g. .com, .org and .net are all top level domain names with unrestricted open domains, whereas .co.uk or .org.uk are secondary level domains (.uk being the secondary level) with open access.

For companies that want to have multiple websites under the same server, there is shared domains, allowing a more economical approach to hosting multiple websites. Shared domains will use a different subdomain with the original domain, for example, Google will have www.google.com, maps.google.com, mail.google.com etc. for each of their different sites.

The rate in which technology is developing and growing into our day-to-day lives is quite rapid compared to 15 years ago. Due to its importance, we would be almost lost without it. A more recent development in remote servers is the cloud, which basically allows users/businesses to store, manage and process data onto external servers rather than locally, meaning that we can access it anywhere (as long as theres an internet connection). Cloud computing has proved to be beneficial in providing better disaster recovery, increased collaboration between users and more environmentally friendly. However, cloud computing does have particular drawbacks, with the main one being that internet access is always required to retrieve from the cloud, meaning that if the internet went down, no-one can access their files. Another major issue is security, as if it is a shared cloud in a businesses for example, it could be easy for the wrong person to get access, or malicious software attacks would be targeted more toward the cloud.

Really Simple Syndication or RSS is a technology that allows you to keep track of updates on websites using a standardised XML format, then shown in an aggregator (reader). RSS feeds allows you to have complete control over what you receive without even entering an email address, meaning no spam, viruses, phishing etc. However, Atom, which is an alternative to RSS was created to address some of the flaws it had, such as differentiating between relative and non-relative URL’s, providing each entry with a globally unique ID, and creating “autodiscovery”, which allows feed aggregators to automatically detect if there is a feed.

This means that it has to be interpreted to include tweets or text messages etc. or to apply to any modern day communication. One of the most common forms of communication for distributed workplaces is mobile technology, as it vastly opens up accessibility as you can use it almost anywhere. This proves most useful when employees are in some downtime, for example, waiting in the airport, or if they are somewhere without access to computers, meaning productivity will increase.

#006 Network Maintenance & Issues

Tuesday 27th September 2016

As we are living through a rapidly growing technological age, we have become reliant, and keep a significant amount of sensitive information digitally. This has directly affected the amount of software attacks from people who are trying to gain access to this information, or delete files from a server. This is why we must protect ourselves against such attacks. Simple ways to help prevent against attacks to your PC or server include keeping your PC’s and antivirus up to date, staying aware of network speed and connectivity, as if it drops or becomes slow it can be due to an attack. For servers, any network ports that aren’t being used should be closed. If an attack were to happen, the first step would be to work out what type of attack, and where its coming from (IP address). If the attack was internal, any devices/PC’s the are affected by the attack should be disconnected, or if it came from outside the network, the IP address should be blocked. Any information/files that are sensitive should be backed up immediately, along with the logs of how the attack happened so it can be prevented against in future. The final step would be to completely reinstall the OS on the affected computers, ensuring that the malware is gone.

However, cyberattacks aren’t the only way that data can be compromised. Another form of attack is social engineering which refers to people purposely tricking someone into giving them sensitive personal details that aren’t available for public viewing. For example, if you were contacted by phone, with someone confirming a few public details (such as name, address etc.) to reel you in, then asking you to confirm a few private details. This form of social engineering is still common, but also exists in fake emails or phishing websites. To help prevent against this, you should always ask the unknown person to confirm who they are and where they’re calling from. Another common form of social engineering is phishing sites, which usually come in the form of an email sent from what appears to be a legitimate business, asking you to provide verification that would then redirect you to a duplicate website to enter your details. It can be more difficult to identify phishing sites, so checking your anti-virus software or security logs for any unsolicited activity, or if you are unsure of the legitimacy of the call/site, its always useful to research into it.

To further protect your data, encryption will often be used. The way encryption works is by using an algorithm to cypher the data, so that if someone were to gain access, it would be meaningless without an encryption key (used to decipher the encryption). There are two main types of encryption: Symmetric and Asymmetric. Symmetric encryption involves just one key, used for both cyphering and deciphering, which poses a security issue, as if someone were to intercept and gain the key, they decrypt the data. This is where Asymmetric encryption has the advantage, as there is a public and a private key in that one key will encrypt the information, whereas the other will decrypt and vice versa. So if the same situation were to occur and someone intercepted the data, they would not be able to decrypt without access to the private key.

The next topic we looked at was Public Key Infrastructure (PKI) which is basically a way in which which the general public can authenticate an entity without sharing any personal details with the website that has requested the authenticity check. The first step is for the entity to request a certificate from the Registration Authority (RA), who will then check the persons details (such as identity or payment methods), and if successful, will then send a request to the Certification Authority (CA). The CA will then send the certificate back to the entity, but will also send the information to the Validation Authority (VA). The purpose of the VA is so that a website can send them the public key (sent by the entity) to confirm their authenticity. However, with having a chain of trust in PKI, there is a liability in that if there was any security issue or if the CA was compromised, the whole PKI is at risk. This happened in 2011, where any certificates given out by Dutch CA DigiNotar were forced to blacklist after there was a number of fake certificates discovered.

Most large companies that are involved with IT and modern technologies will provide their employees with company devices, such as computers or phones, so that they can be easily monitored and controlled. However, some companies will implement a Bring your own device (BYOD) policy, which allows employees to bring their own devices to work, and are permitted to use them for company information and applications. The policy both benefits the company and the employee, as it would save a great deal of costs for the company, but also gives employees better enjoyment of using their own personal device. There are requirements for BYOD, as working with privileged information can be risky with an unknown device. The first requirement is a software application that can manage all the devices on the network, but also ensuring that no device is being misused or unsecured. This is where the second requirement comes in, which is a written policy for both, the employer and employee, outlining their responsibilities, safety and misuse. They will be required to then sign an agreement stating that they have read and understood the policy.

For companies that will have several devices on a network, it can become difficult in making sure that they are all up-to-date and running the same settings. MDM or Mobile Device Management refers to managing and configuring multiple devices across a network, for example, Apple Configurator, which easily allows you to prepare, deploy and supervise iOS devices. MDM proves to be useful for monitoring a device’s life cycle, ensuring that the devices are getting full, efficient use. The life cycle consists of five main stages: Provisioning, Using, Maintaining, Monitoring and Decommissioning. The provisioning stage of the life cycle refers to setting up the device with software, access requirements and networks. The next stage is Using, which is deploying the devices to the users. After a certain amount of time of usage, lets say a year or example, the devices would need security or software updates applied across all the devices, which comes under the Maintaining stage. The Monitoring stage refers to the MDM taking usage data, such as when its used or what its used for, or, for example, in a company that would have delivery systems, the devices could be used for GPS to keep track of where they are or the duration of the journey. And finally, decommissioning, so when the devices are at the end of their life cycle, i.e. become outdated or obsolete, they will be wiped and taken apart/recycled.

In more recent years, cloud-based services are being used more and more throughout our homes and in businesses. it offers many benefits as opposed to locally storing your data, and provides an almost untouchable back if any of the local computers/servers were damaged. However, this service does pose security threats. As cloud servers hold a vast amount of sensitive data, such as financial info, intellectual property, trade secrets etc. they effectively become a prime target for hackers. This consequently leads onto another threat of permanent data loss, as malicious hackers have been known to hack into cloud data centres and permanently delete data to harm businesses or even as extreme as economic damage. DOS or denial-of-service attacks can often occur against cloud services, which basically means that they are overloading the processing power, ultimately crashing the system. However, some of these attacks can often come from inside the cloud service, i.e. a former employee, system administrator or business partner that would target their former workplace as revenge or just having a malicious agenda. There are many other security threats against cloud-based services, such as account hijacking, hacked API’s or system vulnerabilities bu

In the unlikely event of a natural disaster, or if the cloud has been hacked, there would need to be a recovery plan to ensure that the data is protected and that it is still available to the public. There are online web services that can monitor and alert you of any downtime, i.e it will ping every 5 seconds (or any given amount of time) to ensure that the cloud is still up and running, so if it goes down, it will send a notification to the user. As the cloud is reliant upon an internet connection, problems and panic will occur if the connection goes down. This is where a backup connection can become useful to switch to, or a redundant connection that will kick in if another fails. If the cloud service was to completely shut down in the event of a natural disaster, it is always useful to have an onsite backup, and to periodically create the backups to ensure that minimal data is lost. Many would argue that this defeats the purpose of the cloud, but it is a safe option to have onsite backups. Companies like Amazon Web Services (AWS) use zones in the case of a natural disaster, in that if, for example, an earthquake occurs and a zone goes down, it will switch to one of their other zones to ensure that users can still access their information.

Continuous Data Protection or CDP would be a necessity amongst businesses, but depending on the business will depict the best technique to use. For any non-critical data within a network, these can be backed up daily, or every few days. If the non critical data were to be backed up continuously with the critical data, it would result in inefficient waste of processing power. For more critical data, they can be backed up continuously, or even transactionally, i.e. anytime a file is changed, it will be immediately updated or if a file is deleted, it can be recovered straight away from the previous backup. This type of backup can be useful when it comes to transactions or for dropbox where files are exchanged.

Most of us have probably experienced the frustration of a lost internet connection, especially when we cannot seem to fix it. If so, there are a few steps you can take to work out exactly where the problem lies. The first setting that should be checked is to see if TCP/IP is enabled (settings > network > advanced > TCP/IP). If so, you should then check if an IP address has been assigned to the computer which should be under the TCP/IP settings. Below the IP address, the router should be checked, in that if there is no IP address under the router, this means that the computer cannot ping the router. In the same settings menu, you should check if the computer can contact a DHCP service under ‘Configure IPv4’. If everything there is operational, using terminal you can check if the computer can perform a DNS lookup using ‘dig example.com’. However, sometimes the problem is beyond our control, as it could be server-side rather than client-side. Server-side basically means that the problem doesn’t lie with your computer, but with the server you’re trying to interact with. This can be identifiable usually by an error message from the server/browser, or if you can access other services on the same device. If the issue is client-side, the easiest way to check is to try access the service under another device or computer, so if it is accessible on other devices, the computer contains the problem.

Keeping your PC clean and not having any excess files/programs can dramatically affect performance (depending on how much), lifespan and reduces the risk of any computer system failures. Simple ways to maintain your computer include performing a disk cleanup and defragmentation, removing any temporary and cached files, cleaning dust from the fan and keeping all the system and security software up to date.

The motherboard of a computer is used for connecting many of the crucial electronic components, such as the CPU or memory, and is printed on a PCB. Storage devices are used to store all the data within a device and an optical disc, such as CD’s or DVD’s are used for storing, installing and backing up data on your computer.

To establish a basic level of security on a PC, we use authentication, which confirms the identity of a user/computer using a set of principles. There are four basic principles that lie within authentication, with each one containing different forms of security. Basic forms of authentication that exist in most systems would be passwords, security questions or security keys, which fall under the Establishing an Identity principle. This would always be the first step in authentication, as if someone cannot confirm an identity, no further access should be granted. Once the identity is confirmed, establishing access, or checking what the user has access to should then be checked. In many systems, different users will be assigned different roles, each having their own level of access. Different levels of access can determine what information the user has access to, or what actions they can perform. Any of these steps however, will not be secured without the third principle of establishing a secure connection. To ensure that the connection is secure, encryption/decryption is used, with more complex encryption algorithms being more secure. The fourth and final principle is periodic, in that the user may be time bound which sets a time limit, session based which will allocate a user with a session ID after verifying credentials, or request based.

The purpose of Remote access protocols are to control the establishment of connections and the transmission of data over WAN links. Some common protocols include Point-to-Point or PPP, which creates dial-up connections between a client and a server or ISP; Internet Protocol Security or IPsec which can be used as a remote access tunnelling protocol to encrypt traffic going over the internet; Point-to-Point tunnelling protocol (PPTP), which is another tunnelling protocol used with VPNs.

If your connection has dropped and you need to troubleshoot the cable and ADSL modem, follow these few steps to ensure it is done successfully. The first step should be to power-cycle the DSL modem by turning it off and unplugging it for around 30 seconds, then plug it back in and turn it on. Once this is completed, your computer should be rebooted. On the modem, there should be a DSL sync light on, which means its connected to the internet. On the computer, the TCP/IP settings should the be checked, along with the LAN light.

There are a number of ways to diagnose Booting problems with your PC, which can often be overlooked leading to the presumption that the computer is broken. Ways to identify booting problems include: nothing showing on the display, but the display works on another computer; the fans or hard drive is spinning up, but still nothing on the display; the bios screen is displaying or if you hear the bios beeps, which all have different codes for different errors.

Disk partitioning is the separation of one hard drive into multiple independent partitions, which can be individually formatted to perform different tasks. The reason that partitioning is important is that it creates a logical space that the OS can address, as well as reducing the risk of any boot problems, e.g. if you keep your data, applications and OS on the same partition and if something happens to the index file, the computer won’t be able to boot up the hard drive or access any of the data. A Network File System or NFS is an application that allows a user to store, update and view files on a network as if it were on the user’s own computer.

#007 Principles of Internet Security

Wednesday 28th September 2016

There are three major types of encryption, that many people will have probably seen before using Wi-Fi, which are Wired Equivalent Privacy (WEP), Wi-Fi Protected Access (WPA) and Wi-Fi Protected Access 2 (WPA2). WEP was one of the first types of security algorithms introduced to 802.11 standard, using a 64-bit key which would be entered as a string of 10 hexadecimal characters, or a 128-bit key for a string of 26 hexadecimal characters. WEP in modern day age would be a less secure option as opposed newer, more complex algorithms. WPA was the replacement for WEP, which generates a new 128-bit key for each data packet, meaning that it prevents against the type of attacks that WEP was susceptible to. Following on from WPA was WPA2 (802.11i standard), which provides much more secure data protection by implementing FIPS 140-2 AES encryption algorithm, providing government grade security.

Web security is probably the most vital aspect to modern technology, as there is always going to me malicious attackers, whose aim is to steal or destroy sensitive data. Different forms of web security exist, each making it a bit safer from such attackers. Authentication, which is the process of determining if someone/something is what they claim, thus providing access if authenticated. It gives better control over the users in a system and provides an extra layer of protection. Digital certificates, issued from the Registration Authority (RA), is basically an electronic ID for a person, computer or organisation using the public key infrastructure (PKI). #011 Principles of Database Concepts

A Database in computing terms is a collection of data sorted and organised into separate tables allowing it to be easily read and to provide better usability in finding particular data. However, they are then separated into two types; relational and non-relational. Relational databases operate by storing information into different tables with a primary key (unique identifier) and if needed, a foreign key (for relating/linking other tables together), allowing for information to be easily retrieved/correlated and returned to the user. The most popular of these are MySQL, Oracle, or Microsoft SQL, mostly being used in large companies. Non-relational databases however refer to data being stored in an unstructured store i.e. without the table/key model, providing a solution to big data problems. For example, companies such as Google or Facebook have a vast number of applications with millions of users online everyday, so using a relational database would not be able to cope with any traffic spikes.

In relational databases, there are what’s called relationships which allow us to create a link between tables, for example, if there was a company database with an employees table and jobs table, there would be a link between the two so that if a user wanted to know who was working specific jobs, they would just perform a query. However, the relationships are then divided into three main types; One-to-Many, One-to-One and Many-to-Many, each used in different scenarios. One-to-Many is the most common form of relationship which defines as allowing a record in one table to be related to many records in another, e.g. a database holding teacher and pupils’ details would be one-to-many as one teacher can have many pupils. In a One-to-One relationship, only one record can be related to one record from another table and vice versa which occurs when the two related fields are both primary keys or are unique, e.g. a passenger and a passport. Many-to-Many relationships relate to records having many matching records in other tables, and vice versa.

Back when relational databases didn’t exist, it would be very difficult to search through large databases for specific information or groups of records. SQL was a query language designed for relational database management systems that allows a user to update or receive data, with standard SQL commands that provide a global standard of working with databases. Common DBMS that use SQL include Oracle, mySQL, Sybase, Access, Microsoft SQL Server etc.

Boolean operators are the symbols we use to connect different search criteria together to either get more refined or broadened results, with the three operators being AND, OR and NOT (&&, ||, !). The AND operator is applied whenever you want to search using all the specified criteria to get more refined results, e.g. in this if statement, if(x<3 && y=true && z>x), all three conditions will be checked and will only run if all three are true. The OR operator is used when we wish to broaden our results, as it will return if any of the criteria are true, even if the others return false, e.g. in this statement, if(x<3 || y=true || z>x), the statement will run if any of the three conditions are true. And finally, the NOT operator will tell the computer to do the opposite, i.e. if(pwordCheck != password) will run the if statement if the pwordCheck variable is NOT equal to the password variable.

In most search engines you can now use boolean operators when searching to give you a more accurate and refined search results. In Google’s search engine, we can use the boolean operators by using the keyword in caps, for example, search1 AND search2 will search for both words on either side of the operator. This works for the other OR and NOT operators, simply by typing in the word in-between the searches. For any websites out there that wish to be found where searched for, using keywords is the connector that give the websites search engine optimisation (SEO), linking any content or pages so that if someone has searched for it, the website will be shown.

Search sites, or search engines, are websites designed to search and aggregate a range of websites based upon the users search criteria. Meta search engines are what we use to essentially collate results from all the top results from all search engines, then apply their own algorithms to the results. A meta search engine will search the entered string and search it into different engines to then compile a series of results from each. This uses whats called a virtual database by running a query on the other engine databases giving the opportunity to find a mixture of popular and obscure websites. To enhance the accuracy and overall performance of search engines, they release spiders or web crawlers to browse the World Wide Web creating a search engine index. Spiders will take it upon themselves to index every word on the page and following every link to ensure that everything is accounted for by taking note of two things in the html: the words within the page and where the words are found. Spiders are very fast travelling, with some able to crawl over 25 pages per second and generate over 150 kbp/s of data, meaning that if a company were to release a large amount of spiders, they could respectively search a large portion of websites to give more accurate search engine results.

Researching into different articles or papers can be a tedious task in that there are thousands available for most subjects. However, some papers may not be completely accurate or can vary from the truth which is why you’ll find papers that are peer-reviewed. There are two different types of peer review: authoritative and open. Authoritative peer review refers to papers that are reviewed by a small group of authors or researchers that will generally be experts in that particular field, but the feedback will be completely anonymous, whereas open peer reviews use a more open group of people, who many not be experts in the field, but would have adequate knowledge to provide some feedback relating to the paper. Open peer reviews are also not anonymous, meaning that the author can see who reviewed the paper.

Gathering research online can be tricky to get relevant or accurate sources which is why different forms of research can be carried out to ensure accurate and relevant research is gathered. Primary resources are obtained “brand-new” in that the information is gathered specific to what is needed directly from primary research, which could include surveys, direct feedback or focus groups. This can be a costly approach to market research as it requires planning, setting up and can be time consuming. For many businesses this may not be an efficient approach, especially if some of the research has been carried out before. This is where secondary resources comes into play as it is resources or research that has been carried out before, such as analysis reports, critique or other market research. This approach is much more cost and time effective but will deprive in the accuracy and meticulousness that primary resources will provide. And finally, indexes or tertiary resources is essentially an address book for primary and secondary resources.

#018 Email in the Workplace

Monday 28th November 2016

Multipurpose Internet Mail Extensions or MIME is an email extension that allows the user to send various file formats such as images, audio or video by formatting non-ASCII messages. S/MIME or Secure Multipurpose Internet Mail Extensions is the standard for encrypting and signing emails that are sent to ensure that the person receiving the email is certain that the message is genuine and not a counterfeit user. PGP (Pretty Good Privacy) is software owned by PGP inc. that uses various encryption methods, such as data compression, hashing, public and symmetric key to secure any data being transferred or communicated, usually used for text files, emails, directories and disk partitions.

An email signature is a section of text that appears at the bottom of any email that is sent, usually containing personal information such as name, contact information, business information or any links. Using this in a business situation is always useful as it provides an easy way for someone to contact you back without looking up the information manually. Another useful feature now used widely in emails is the conversation threads which systematically groups any messages being sent back and fourth from recipients. This proves useful to people who would send and receive numerous emails daily as it would become difficult to keep track of every conversation or remember what the topic of discussion is about. Using this feature can prove useful to many, but most of all to people who have high email traffic or to those who have a busy schedule and don’t have time to be looking back on previous emails.

BCC or Blind Carbon Copy is an email feature that allows the sender to send an email to someone with the identity protected from the other primary and secondary recipients (To and CC). In any email client, there will be a forward and a reply option, usually with quite similar buttons, but perform two different tasks. If a user forwards an email, it essentially sends a copy of the received email to a different recipient whereas replying is simply sending an email back to the person who sent the original message. Emails are commonly used in the workplace to communicate to multiple users, transfer files or information, arranging meetings or communicating between customers or other businesses. The content of the email should always be concise, with a clear logical order and a direct subject line to capture the attention of the recipient. Any emails being sent in the workplace should always have a professional tone with proper use of grammar and punctuation, then followed by a closing signature at the bottom, usually ending with “regards” or “yours sincerely” along with the users name. However, using emails are targets for spreading viruses or subject to phishing as many businesses will transfer useful or sensitive information. Viruses can be spread simply by opening a link or attachment which will be sent by a hoax user. A release of a virus in a local business network could be catastrophic as it could compromise valuable company information or even shut down the internal network altogether.
