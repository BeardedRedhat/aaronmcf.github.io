Internet Architecture

Tuesday 20th September 2016 - This unit is based around the Internet architecture and the various types of networks, bandwidths, protocols etc. that is needed to connect a client to the internet. A client server network is designed to allow the client access to resources from a central computer, i.e a server. The main purpose of this is that it acts as a hub for a number of less powerful computers. The hardware and software that a client needs to successfully connect to the internet is a PC or a device connected to a network card or modem, which will be connected to a router that sends and receives the data packets, and a suitable browser on the device so that it can interpret the data using protocols and display it to the screen.

The data is transmitted from the client to the server can vary in speed, in both upload and download times. This is relative to the bandwidth, which is the amount of data that can be transmitted at any given time (usually measured per second). The three types of bandwidth that we looked at was PSTN, ISDN and DSL. PSTN or Public Switched Telephone Network is basically the world’s circuit-switched telephone networks, sometimes referred to as the Plain Old Telephone Service (POTS). ISDN or Integrated Services Digital Network is used for sending voice, video and data over digital or normal telephone lines, which supports transfer rates of around 64 Kbps/s. DSL or Digital Subscriber Line was created to allow high bandwidth transfer rates across all homes and businesses using just normal telephone lines. The speed can range anywhere from 256kbit/s to over 100Mbit/s.

When data is transferred from server to the client, it will be just raw data which would be meaningless without protocols. There are various different types of Protocol, such as TCP/IP, HTTP, SMTP or POP3, which are made up of rules so that computers can interpret and read data. Transmission Control Protocol (TCP) is the process of breaking up information that has been sent into smaller pieces, called data packets. These packets are then sent and put back together in the correct order. Internet Protocol (IP) is linked with TCP as it is the final stage in taking the data packets and directing them to the required destination, using an IP address which will be unique to every computer. Hypertext Transfer Protocol or HTTP is most commonly known for being the protocol used for the World Wide Web. It is the process of using TCP/IP to transfer HTML, so that when a URL is entered into a browser, it will fetch and display the webpage. Simple Mail Transfer Protocol (SMTP) and Post Office Protocol version 3 (POP3) are used for the sending and receiving of electronic mail. SMTP is used mainly in the sending of emails, whereas POP3 is used along side to receive emails, as SMTP has a limited ability to queue incoming messages.

There is a process of how the internet works, and how we receive data transmissions. It begins with a PC or device that is linked to a router. If, for example, the client enters a URL into a browser, it will send a request to the closest Point of Presence, which is run by your Internet service provider. It will then send back the necessary data back to the router/modem. If, however, you require information from another country or ISP, it will then find the closest Internet Exchange Point (IXP), which will cover several thousand POP’s. It is then sent back to the POP, then back to the client. This process can result in longer loading times, but in reality, is a very efficient way of networks to exchange information, as well as reducing costs.

When a user has entered a domain to take them to a website, the IP address needs to be located. This is where Domain Name Systems (DNS) comes into play as it acts as an address book for IP’s, making it an essential component of the internet. The way that an IP address will be located is by the naming structure, i.e, the prefix (http) tells us that we want to use the Hypertext transfer protocol and that we are communicating with the World Wide web. The subdomain (www.) also tells us that we are searching the WWW. The domain name is the next part (example.com), which is the website unique identifier, as no two websites will have the same domain. The last part of the domain name (.com) is called the extension, which tells us what type of website is being run, e.g. .com, .org and .net are all top level domain names with unrestricted open domains, whereas .co.uk or .org.uk are secondary level domains (.uk being the secondary level) with open access.

For companies that want to have multiple websites under the same server, there is shared domains, allowing a more economical approach to hosting multiple websites. Shared domains will use a different subdomain with the original domain, for example, Google will have www.google.com, maps.google.com, mail.google.com etc. for each of their different sites.

The rate in which technology is developing and growing into our day-to-day lives is quite rapid compared to 15 years ago. Due to its importance, we would be almost lost without it. A more recent development in remote servers is the cloud, which basically allows users/businesses to store, manage and process data onto external servers rather than locally, meaning that we can access it anywhere (as long as theres an internet connection). Cloud computing has proved to be beneficial in providing better disaster recovery, increased collaboration between users and more environmentally friendly. However, cloud computing does have particular drawbacks, with the main one being that internet access is always required to retrieve from the cloud, meaning that if the internet went down, no-one can access their files. Another major issue is security, as if it is a shared cloud in a businesses for example, it could be easy for the wrong person to get access, or malicious software attacks would be targeted more toward the cloud.

Really Simple Syndication or RSS is a technology that allows you to keep track of updates on websites using a standardised XML format, then shown in an aggregator (reader). RSS feeds allows you to have complete control over what you receive without even entering an email address, meaning no spam, viruses, phishing etc. However, Atom, which is an alternative to RSS was created to address some of the flaws it had, such as differentiating between relative and non-relative URL’s, providing each entry with a globally unique ID, and creating “autodiscovery”, which allows feed aggregators to automatically detect if there is a feed.
